## 命名规则

1. 文件名格式: `{Prefix}{N}-{中文标题}.md`
   - 例: `B8-空集守卫.md`(新 B 条从 B8 起,不重用 B2-B4 因为 DYNAMIC 区有 patch,留待晋升)
2. N 数字递增规则: **同前缀下取 max(已占用数字) + 1**,从 1 开始
3. 中文标题:
   - 来源:`draft.key_concepts[0].title`(LLM 提取的第一条核心概念)
   - 截断到 ≤ 12 个汉字
   - 过滤 Windows 非法字符: `\/:*?"<>|`(全部替换为 `-`)
   - 过滤控制字符(0x00-0x1f, 0x7f)
4. 角色索引 primitive 文件(在 `角色技能/se/` 根,**不**是 skill 文件,不要覆盖):
   - `F-前端.md / F-后端.md / F-技术主管.md / F-架构.md` — 这是角色索引,**不要** 改/移/重命名

## 冲突检测

每次 route 前 scan `20-知识\角色技能\se\{角色子目录}\` 下 `^{Prefix}\d+-` 正则,计算 max N,新条目 = max N + 1。

```python
# 示例: 给后端追加新 B* skill
existing = list(Path("20-知识/角色技能/se/后端工程师").glob("B*-*.md"))
nums = [int(p.stem[1:].split("-")[0]) for p in existing]  # [1, 5, 6, 7]
next_n = max(nums) + 1  # 8
new_file = f"B{next_n}-新标题.md"
```

## 角色基因 frontmatter 注入格式

### skill_refs 追加项

```yaml
skill_refs:
  - 20-知识/角色技能/se/{后端工程师}/{新文件名}.md
```

注意:
- 用 vault 相对路径(`D:\Markdown\memory\adam\` 为根)
- 用正斜杠(`/`)不用反斜杠
- `os.path.normpath` 去重,避免大小写/斜杠差异导致的重复

### rule_refs 追加项

```yaml
rule_refs:
  - '[[{新文件名}#3. 核心约束]]'
```

注意:
- 双中括号 `[[]]` 是 Obsidian wikilink
- `#3. 核心约束` 是 vault skill 文件的章节锚点(若 LLM 没生成这个章节,锚点失效但 wikilink 仍生效)
- 单引号包裹,避免 YAML 解析 `[[` 开头的字符串
